Detailed Description

A surfaceWriter for STARCD files.

The formatOptions for starcd:

Property Description Required Default
compression Use file compression no false
scale Output geometry scaling no 1
transform Output coordinate transform no
fieldLevel Subtract field level before scaling no empty dict
fieldScale Output field scaling no empty dict

The geometry is written via the MeshedSurfaceProxy, the fields are written in a trivial ASCII format with ID and VALUE as so-called user data. These .usr files can be read into proSTAR with these types of commands. For element data: 

        getuser FILENAME.usr cell scalar free
        getuser FILENAME.usr cell vector free

and for vertex data: 

        getuser FILENAME.usr vertex scalar free
        getuser FILENAME.usr vertex vector free

file locations

The rootdir normally corresponds to something like postProcessing/<name>

Geometry

    rootdir
    `-- timeName
        `-- surfaceName.{cel,vrt,inp}

Fields

    rootdir
    `-- timeName
        |-- <field0>_surfaceName.{usr}
        `-- <field1>_surfaceName.{usr}
