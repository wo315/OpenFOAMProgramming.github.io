Detailed Description

template<class CloudType>
class Foam::ParticleHistogram< CloudType >

Computes a histogram for the distribution of particle diameters and corresponding number of particles hitting on a given list of patches or face zones.

Operands:

Operand Type Location
input - -
output file dat <case>/postProcessing/<FO>/<time>/<file>
output field - -

The output file contains four columns, from the first to the last:

= Left bin-edge of particle diameter
= Right bin-edge of particle diameter
= Number of particles falling into the corresponding bin
= Cumulative number of binned particles
Usage
Minimal example by using constant/reactingCloud1Properties.cloudFunctions: 
particleHistogram1
{
    // Mandatory entries
    type                 particleHistogram;
    nBins                <label>;
    min                  <scalar>;
    max                  <scalar>;
    maxStoredParcels     <scalar>;

    // Conditional entries

        // Option-1
        patches          (<wordRes>);

        // Option-2
        faceZones        (<wordRes>);

    // Inherited entries
    ...
}

where the entries mean:

Property Description Type Reqd Deflt
type Type name: particleHistogram word yes -
nBins Number of histogram bins label yes -
max Maximum value of histogram data scalar yes -
min Minimum value of histogram data scalar yes -
maxStoredParcels Maximum number of parcels to process label yes -
patches Names of operand patches wordRes choice -
faceZones Names of operand face zones wordRes choice -

The inherited entries are elaborated in:

  • CloudFunctionObject.H
  • writeFile.H
Note
  • The underlying type of maxStoredParcels is set as a scalar for I/O.
